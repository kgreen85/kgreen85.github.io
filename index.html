Adaptation
Accuracy Report

The Adaptation Accuracy Report is a site for rating how accurate a movie or tv show is to its source material. The site was created by myself and 3 other developers. My primary role was the backend and deployment. I also assisted with front-end development.
The site is monolith built with Django running in Docker. The backend is built using RESTful API's to dynamically serve the React frontend. A 3rd party api was used to display the movie description, IMDB rating and movie poster. We set up a search function to allow users to search the api and add movies to our PostgresSQL database. The site also includes user creation and login. The site was deployed to Heroku using the Gitlab CI/CD pipeline. A link to the site is included below.

Conference Go

This is an app for creating and attending conferences. It features the ability to create a new conference and its location, sign up presenters with the ability for the admin to approve or deny the presentation with automated email response and sign up to attend a conference.

The app is built on Django with a RESTful API backend. It also uses two 3rd party api's run trough an anti-corruption layer. The api's are used to dynamically generate the conference picture and current weather data based on location. The front-end is build using JavaScript and React.

CarCar

CarCar is a car dealership application for managing inventory, service appointments and sales. This was a project that I worked with another developer on. I was responsible for the service end of the app as well as the inventory portion..

The Application is built with Django using RESTful API endpoints. It is broken up into 3 microservices that share data from PostgresSQL database via a polling. The app runs in Docker. Backed was built using React to create a responsive Single Page Application (SPA).
